# Code of Conduct

This project adopts the **Contributor Covenant, version 2.1**, available at https://www.contributor-covenant.org/version/2/1/code_of_conduct/.

The Contributor Covenant is itself licensed under CC-BY 4.0 and is used here unmodified. The full text at the link above is the authoritative version; this file summarizes the project's commitment and reporting process.

## Our commitment

We are committed to making participation in this project a respectful experience for everyone. We welcome contributors from any background, experience level, nationality, identity, or perspective, and we expect everyone involved — maintainers, contributors, and users — to help keep the community welcoming and constructive.

## Expected behavior

- Use welcoming and inclusive language.
- Respect differing viewpoints and experiences, especially around language use, nuance, and regional / dialectal variation. This project is about Japanese as a living language, and there is room for disagreement.
- Focus criticism on the work (data, code, decisions), not the person.
- Prefer concrete examples and references when pointing out errors.
- Assume good intent, and ask before assuming the worst.
- Credit contributors where credit is due — especially native speakers and reviewers whose work improves the dataset.

## Unacceptable behavior

- Personal attacks, insults, or demeaning comments.
- Discrimination based on any protected characteristic.
- Trolling, deliberate intimidation, or sustained disruption of discussions.
- Publishing others' private information without permission.
- Other conduct that a reasonable participant would consider unprofessional or unwelcome.

The full Contributor Covenant lists these and other expectations in more detail. When in doubt, defer to the linked document.

## Scope

This Code of Conduct applies to all project spaces — the GitHub repository (issues, pull requests, discussions), commit messages, the `docs/` tree, and any project-related communication channel that may be established (mailing lists, chat, etc.). It also applies when a contributor is representing the project in public.

## Reporting

If you witness or experience conduct that violates this Code of Conduct, please report it to the project maintainer:

- **GitHub**: open a **private** security advisory at https://github.com/jkindrix/japanese-language-data/security/advisories/new, or
- **Email**: jkindrix@gmail.com

Reports will be handled confidentially. Please include:

- What happened (as specifically as you are comfortable sharing)
- Where and when (issue number, PR number, commit, or date/time for ephemeral channels)
- Who was involved (usernames / handles)
- Any prior context that helps assess the situation

The maintainer will acknowledge receipt within a reasonable time and will decide on a response after investigation. Possible responses range from a private conversation to temporary or permanent removal from the project, depending on severity and context.

## Enforcement

The Contributor Covenant v2.1 includes "Enforcement Guidelines" that describe a graduated response (correction → warning → temporary ban → permanent ban). This project follows those guidelines when responding to violations.

A maintainer who receives a report is expected to:

1. Acknowledge the report promptly.
2. Treat the reporter and the subject of the report fairly — this is not an adversarial process, it is a community-health process.
3. Take action proportional to the violation.
4. Document the outcome so that patterns can be tracked if necessary.

## Appeals

If you believe an enforcement action was incorrect, you can appeal by contacting the maintainer through the same reporting channels. Appeals will be considered on their merits; the maintainer will explain the decision.
